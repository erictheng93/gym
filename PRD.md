## 1. 專案概述 (Overview)

### 1.1 背景

目前的健身房管理軟體多為舊式單體架構，難以擴充且缺乏現代化的使用者體驗。常見痛點包括：

- **資料孤島**：各分店數據無法統一彙整，總部難以掌握全局
- **人工作業繁瑣**：合約請假/轉讓需人工計算到期日，易出錯
- **權限管理粗糙**：無法細緻控制「教練只能看自己學員」等場景
- **報表滯後**：財務數據需事後人工統計，無法即時決策
- **會員體驗差**：缺乏自助服務 App，預約/查詢需臨櫃處理
- **教練管理分散**：執課紀錄、學員追蹤無統一系統

### 1.2 解決方案

構建一套支援 **多場館 (Multi-branch)**、**高度自動化** 且 **數據驅動** 的 SaaS 系統：

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        Gym Nexus 系統架構                                │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌────────┐│
│  │ 會員端  │ │ 教練端  │ │ 櫃檯端  │ │ 店長端  │ │ 行銷端  │ │ HR端   ││
│  │  App    │ │  App    │ │  Web    │ │  Web    │ │  Web    │ │  Web   ││
│  └────┬────┘ └────┬────┘ └────┬────┘ └────┬────┘ └────┬────┘ └───┬────┘│
│       │          │          │          │          │          │        │
│       └──────────┴──────────┴────┬─────┴──────────┴──────────┘        │
│                                  │                                     │
│                          ┌───────▼───────┐                            │
│                          │   Directus    │                            │
│                          │   (API/CMS)   │                            │
│                          └───────┬───────┘                            │
│                                  │                                     │
│                          ┌───────▼───────┐                            │
│                          │  PostgreSQL   │                            │
│                          │   (RLS 權限)  │                            │
│                          └───────────────┘                            │
└─────────────────────────────────────────────────────────────────────────┘
```

### 1.3 目標

| 類型 | 目標 | 衡量指標 |
|------|------|----------|
| **擴展性** | 支援 50+ 間分店數據量 | 報表查詢 < 2 秒 |
| **自動化** | 合約異動自動計算到期日 | 人工錯誤率 < 1% |
| **精準權限** | 多層權限分級 (7 種角色) | 資料外洩事件 = 0 |
| **即時報表** | 財務/HR/行銷自動統計 | 報表產出時間 < 1 分鐘 |
| **會員體驗** | App 自助服務 | 臨櫃作業減少 50% |
| **教練效率** | 統一執課管理 | 紀錄完整率 > 95% |

### 1.4 範圍界定

#### In Scope (MVP + Phase 2)
- 多分店組織架構與權限
- 會員 CRUD 與標籤系統
- 合約生命週期管理 (含請假/轉讓)
- 財務收款紀錄與報表
- HR 打卡、排班、休假、薪資
- **會員端 App** (預約、查詢、目標追蹤)
- **教練端 App** (學員管理、執課紀錄)
- **行銷後台** (Leads、活動、RFM)
- **人資後台** (考核、升遷、薪資管理)

#### Out of Scope (Future)
- Wger 運動數據深度整合
- 進階 BI 分析 (AI 預測)
- Mobile Native App (暫用 PWA)
- 第三方金流串接

## 3. 功能需求 (Functional Requirements)

### 3.1 組織與權限模組 (Organization & Access)

#### FR-ORG-001: 多分店管理
| 項目 | 說明 |
|------|------|
| **描述** | 系統需支援新增分店，並設定分店屬性 |
| **分店類型** | `HEADQUARTER` (總店) / `BRANCH` (分店) |
| **分店屬性** | 名稱、地址、統編、營業時間、特定設定 (JSON) |
| **限制** | 總店僅能有一間，分店數量無上限 |

#### FR-ORG-002: 職位權限配置
| 項目 | 說明 |
|------|------|
| **描述** | 透過 `job_titles` 表定義職位及其預設權限 |
| **權限格式** | JSON 格式的 `permissions_config` 欄位 |
| **UI 呈現** | 前端以 Checkbox 方式呈現可勾選權限 |
| **覆寫機制** | 員工層級可透過 `custom_permissions` 覆寫職位預設 |

#### FR-ORG-003: 資料隔離 (Row-Level Security)
| 項目 | 說明 |
|------|------|
| **描述** | 分店員工預設只能存取 `branch_id` 與自己相同的資料 |
| **實作** | Directus Policy + PostgreSQL RLS |
| **規則** | 見下方權限規則表 |

**權限規則：**
| 角色 | 資料範圍 | Policy 條件 |
|------|----------|-------------|
| HQ_ADMIN | 全部 | `true` |
| STORE_MANAGER | 本店 | `branch_id = $CURRENT_USER.branch_id` |
| COACH | 負責會員 | `sales_person_id = $CURRENT_USER.id` |
| MARKETING | 全部會員 (唯讀) | `true` (read only) |
| HR | 本店員工 | `branch_id = $CURRENT_USER.branch_id` |

### 3.3 合約與會籍系統 (Contracts & Plans)

#### FR-CON-001: 方案定義
| 項目 | 說明 |
|------|------|
| **期限制** | `TIME_BASED` - 月費/年費，有明確到期日 |
| **點數制** | `COUNT_BASED` - 課程包，有剩餘堂數 |
| **方案屬性** | 名稱、價格、期限/堂數、是否允許請假、是否允許轉讓 |

#### FR-CON-002: 電子合約生命週期

```
┌─────────┐     ┌─────────┐     ┌─────────┐     ┌─────────┐
│  草稿   │────►│  有效   │────►│  暫停   │────►│  過期   │
│ (DRAFT) │     │(ACTIVE) │     │(PAUSED) │     │(EXPIRED)│
└─────────┘     └────┬────┘     └────┬────┘     └─────────┘
                     │               │                ▲
                     │               └────────────────┤
                     │                                │
                     └──────────────────────────────►─┘
                                 解約 (TERMINATED)
```

| 狀態 | 說明 | 觸發條件 |
|------|------|----------|
| DRAFT | 業務建立，待簽名 | 新建合約 |
| ACTIVE | 簽名並付款後生效 | 完成簽名 + 付款 |
| PAUSED | 會員請假中 | 建立請假紀錄 |
| EXPIRED | 超過到期日 | `end_date < NOW()` |
| TERMINATED | 提前解約 | 手動操作 |

#### FR-CON-003: 合約異動邏輯

| 異動類型 | 說明 | 對 `end_date` 的影響 |
|----------|------|---------------------|
| **請假 (Pause)** | 暫停會籍 | 順延 = 請假天數 |
| **轉讓 (Transfer)** | 變更會員歸屬 | 不變 |
| **延期 (Extension)** | 手動展延 | 增加指定天數 |

#### FR-CON-004: 電子簽名與 PDF
| 項目 | 說明 |
|------|------|
| **簽名方式** | Canvas 手寫簽名 |
| **儲存格式** | PNG 圖檔，存於 Directus Files |
| **PDF 生成** | 合約簽署後自動生成 PDF 存檔至 R2 |
| **內容包含** | 合約條款、會員資料、簽名圖檔、簽署時間戳 |

#### FR-CON-005: 續約提醒
| 項目 | 說明 |
|------|------|
| **提醒時機** | 到期前 30 天、14 天、7 天 |
| **通知管道** | App 推播、簡訊、Email |
| **對象** | 會員本人 + 負責業務/教練 |

## 7. 驗收標準 (Acceptance Criteria)

### AC-001: 權限隔離
| 條件 | 預期結果 |
|------|----------|
| 店長登入系統 | 只能看到自己分店的會員與報表 |
| 教練登入系統 | 只能看到自己負責的會員 |
| 總部登入系統 | 可看到所有分店數據 |
| 教練嘗試修改價格 | 系統拒絕，顯示權限不足 |
| 行銷嘗試查看薪資 | 系統拒絕，顯示權限不足 |

### AC-002: 合約請假順延
| 條件 | 預期結果 |
|------|----------|
| 建立一年期合約 (到期日 2025-12-31) | 系統顯示 `end_date = 2025-12-31` |
| 設定請假 30 天 (2025-03-01 ~ 2025-03-30) | 系統自動更新 `end_date = 2026-01-30` |
| 再設定請假 15 天 | 系統自動順延 `end_date` 15 天 |

### AC-003: HR 打卡
| 條件 | 預期結果 |
|------|----------|
| 員工打卡 Check-in 09:00, Check-out 18:00 | 系統顯示工時 = 9 小時 |
| 員工未打卡 | 系統標記該日為「異常」 |

### AC-004: 約課系統
| 條件 | 預期結果 |
|------|----------|
| 會員預約教練課程 | 預約成功，扣除 1 堂 (待執行) |
| 課程完成 | 執課紀錄建立，堂數正式扣除 |
| 會員取消課程 (24 小時前) | 取消成功，堂數歸還 |
| 會員取消課程 (2 小時前) | 系統提示需扣堂或收費 |

### AC-005: 教練執課紀錄
| 條件 | 預期結果 |
|------|----------|
| 教練建立執課紀錄 | 紀錄成功儲存 |
| 教練嘗試刪除紀錄 | 系統拒絕，提示需主管審核 |
| 店長審核刪除 | 刪除成功 |

### AC-006: Lead 轉換追蹤
| 條件 | 預期結果 |
|------|----------|
| 新 Lead 從 FB 廣告進入 | 來源標記為 FB，狀態為「新進」 |
| Lead 成交為會員 | 狀態更新，可追蹤轉換來源 |
| 查看轉換報表 | 顯示各管道轉換率 |

### AC-007: 會員 App 功能
| 條件 | 預期結果 |
|------|----------|
| 會員查看合約 | 顯示所有合約，含到期日、剩餘堂數 |
| 會員預約課程 | 顯示可約時段，預約成功 |
| 會員設定健身目標 | 目標儲存，可追蹤進度 |
