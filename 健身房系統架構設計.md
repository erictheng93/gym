### 2.2 關聯說明

| 關聯 | 類型 | 說明 |
|------|------|------|
| branches → employees | 1:M | 一間分店有多位員工 |
| job_titles → employees | 1:M | 一個職位可指派給多位員工 |
| branches → members | 1:M | 一間分店有多位會員 (主場館) |
| employees → members | 1:M | 一位業務負責多位會員 |
| members → contracts | 1:M | 一位會員可有多張合約 |
| membership_plans → contracts | 1:M | 一種方案可產生多張合約 |
| contracts → contract_logs | 1:M | 一張合約可有多筆異動 |
| contracts → payments | 1:M | 一張合約可有多筆收款 |
| employees → attendances | 1:M | 一位員工有多筆打卡紀錄 |
| employees → leave_requests | 1:M | 一位員工可申請多筆休假 |

## 3. 詳細資料表結構 (Database Schema)

### A. 組織與權限模組 (Organization & Access)

#### 3.1 branches (分店/場館)

> 管理總店與分店資訊，作為資料隔離的基礎。

| 欄位 | 類型 | 必填 | 預設值 | 說明 |
|------|------|:----:|--------|------|
| `id` | UUID | ✅ | auto | 主鍵 |
| `status` | String | ✅ | 'published' | Directus 狀態 |
| `name` | String(100) | ✅ | - | 分店名稱 (e.g., 太原店) |
| `type` | Enum | ✅ | 'BRANCH' | `HEADQUARTER` / `BRANCH` |
| `code` | String(20) | ✅ | - | 分店代碼 (e.g., TAY) |
| `address` | String(255) | ❌ | - | 地址 |
| `phone` | String(20) | ❌ | - | 聯絡電話 |
| `tax_id` | String(20) | ❌ | - | 統一編號 |
| `settings` | JSON | ❌ | `{}` | 分店特定設定 |
| `created_at` | DateTime | ✅ | now() | 建立時間 |
| `updated_at` | DateTime | ✅ | now() | 更新時間 |

**settings JSON 結構範例：**
```json
{
  "business_hours": {
    "weekday": { "open": "06:00", "close": "23:00" },
    "weekend": { "open": "08:00", "close": "22:00" }
  },
  "default_contract_terms": "本合約依據...",
  "allow_cross_branch_access": true
}
```

**索引：**
- `idx_branches_type` ON `type`
- `idx_branches_code` ON `code` (UNIQUE)

#### 3.7 contract_logs (合約異動紀錄)

> 處理會籍暫停、延期、轉讓等異動。

| 欄位 | 類型 | 必填 | 預設值 | 說明 |
|------|------|:----:|--------|------|
| `id` | UUID | ✅ | auto | 主鍵 |
| `contract_id` | UUID (FK) | ✅ | - | → `contracts.id` |
| `type` | Enum | ✅ | - | `PAUSE` / `EXTENSION` / `TRANSFER` |
| `start_date` | Date | ✅ | - | 開始日期 |
| `end_date` | Date | ✅ | - | 結束日期 |
| `days` | Integer | ✅ | - | 天數 (自動計算) |
| `reason` | String(255) | ❌ | - | 異動原因 |
| `old_member_id` | UUID (FK) | ❌ | - | 轉讓前會員 (TRANSFER) |
| `new_member_id` | UUID (FK) | ❌ | - | 轉讓後會員 (TRANSFER) |
| `created_by` | UUID (FK) | ✅ | - | → `employees.id` |
| `created_at` | DateTime | ✅ | now() | 建立時間 |

**type 類型說明：**
| 類型 | 說明 | 對 end_date 影響 |
|------|------|------------------|
| `PAUSE` | 請假暫停 | 順延 days 天 |
| `EXTENSION` | 手動延期 | 順延 days 天 |
| `TRANSFER` | 轉讓 | 不變，變更會員 |

**請假順延 SQL 邏輯：**
```sql
-- 更新合約到期日
UPDATE contracts
SET end_date = original_end_date + (
    SELECT COALESCE(SUM(days), 0)
    FROM contract_logs
    WHERE contract_id = contracts.id
    AND type IN ('PAUSE', 'EXTENSION')
)
WHERE id = :contract_id;
```

```sql
CREATE OR REPLACE FUNCTION update_contract_paid_amount()
RETURNS TRIGGER AS $$
BEGIN
    UPDATE contracts
    SET paid_amount = (
        SELECT COALESCE(SUM(
            CASE WHEN type = 'INCOME' THEN amount
                 WHEN type = 'REFUND' THEN -amount
            END
        ), 0)
        FROM payments
        WHERE contract_id = NEW.contract_id
    ),
    payment_status = CASE
        WHEN paid_amount >= total_amount THEN 'PAID'
        WHEN paid_amount > 0 THEN 'PARTIAL'
        ELSE 'UNPAID'
    END
    WHERE id = NEW.contract_id;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;
```

### 4.3 前端動態權限實作

**Nuxt Middleware 範例：**
```typescript
// middleware/permission.ts
export default defineNuxtRouteMiddleware(async (to) => {
  const { user } = useAuth()
  const permissions = await getEffectivePermissions(user.value)

  // 檢查路由所需權限
  const requiredPermission = to.meta.permission
  if (requiredPermission && !permissions[requiredPermission]) {
    return navigateTo('/403')
  }
})

// 合併職位預設 + 個人覆寫
async function getEffectivePermissions(user) {
  const employee = await fetchEmployee(user.id)
  const jobTitle = await fetchJobTitle(employee.job_title_id)

  return {
    ...jobTitle.permissions_config,
    ...employee.custom_permissions  // 覆寫
  }
}
```

## 5. 報表邏輯 (Reporting Queries)

### 5.1 日報表 - 新合約數

```sql
SELECT
    COUNT(*) as new_contracts,
    SUM(total_amount) as total_revenue
FROM contracts
WHERE DATE(created_at) = CURRENT_DATE
AND branch_id = :branch_id
AND status != 'DRAFT';
```

### 5.2 應收帳款 (AR) 列表

```sql
SELECT
    c.contract_no,
    m.full_name as member_name,
    c.total_amount,
    c.paid_amount,
    (c.total_amount - c.paid_amount) as outstanding
FROM contracts c
JOIN members m ON c.member_id = m.id
WHERE c.payment_status = 'PARTIAL'
AND c.branch_id = :branch_id
ORDER BY outstanding DESC;
```

### 5.3 教練業績統計

```sql
SELECT
    e.full_name as coach_name,
    COUNT(c.id) as contract_count,
    SUM(c.total_amount) as total_revenue
FROM contracts c
JOIN employees e ON c.sales_person_id = e.id
WHERE c.created_at BETWEEN :start_date AND :end_date
AND c.branch_id = :branch_id
GROUP BY e.id, e.full_name
ORDER BY total_revenue DESC;
```

### 5.4 會員狀態分佈

```sql
SELECT
    status,
    COUNT(*) as count
FROM members
WHERE branch_id = :branch_id
GROUP BY status;
```

### 5.5 合約到期預警 (30 天內)

```sql
SELECT
    c.contract_no,
    m.full_name,
    m.phone,
    c.end_date,
    (c.end_date - CURRENT_DATE) as days_remaining
FROM contracts c
JOIN members m ON c.member_id = m.id
WHERE c.status = 'ACTIVE'
AND c.end_date BETWEEN CURRENT_DATE AND (CURRENT_DATE + INTERVAL '30 days')
AND c.branch_id = :branch_id
ORDER BY c.end_date;
```

### 6.2 報表用 Materialized View (建議)

```sql
-- 每日營收彙總 (每小時刷新)
CREATE MATERIALIZED VIEW mv_daily_revenue AS
SELECT
    DATE(payment_date) as date,
    branch_id,
    SUM(CASE WHEN type = 'INCOME' THEN amount ELSE -amount END) as revenue,
    COUNT(*) as transaction_count
FROM payments
GROUP BY DATE(payment_date), branch_id;

-- 刷新
REFRESH MATERIALIZED VIEW mv_daily_revenue;
```
